# scripts/ops

一次性 / 手动运维脚本的**存档**。

**这些不是应用代码 —— `deploy.sh` 不会执行它们。** 放在这里只为可追溯：记录「在生产服务器上到底跑过什么、为什么、怎么回滚」。

## 约定

- 每个脚本自带完整头注释（为什么这么改、怎么跑、如何回滚）。
- 需要 root 的脚本会明确标注。本仓库的 `deploy` 用户**不能免密 sudo**、root 也不给 SSH 公钥登录，故这类脚本须由持 root 者手动执行（阿里云控制台「远程连接」开 root shell，或 `sudo bash <脚本>`）。
- **已执行过的脚本按原样存档，不为「看起来更整洁」而事后改写** —— 改了就对不上当时真正跑过的版本。若逻辑有 bug，修复本身也作为一次带说明的提交，而非静默覆盖。
